# ============================================================
# OPD 训练 + 评测使用说明（对应论文 Table 2 的 OPD baseline）
# ============================================================
# OPD = on-policy distillation，仅 clipped reverse KL（不含 EOPD 的熵门控 forward-KL）。
# 与 EOPD 的区别【仅 METHOD 一个开关】：
#   - EOPD : METHOD=eopd  -> 额外在教师高熵 token 上加 entropy-gated forward-KL
#   - OPD  : METHOD=opd   -> 只保留 clipped reverse KL（baseline）
# 两者共用同一套脚本、同一份训练数据、同一对学生/教师模型，只差这一个 flag。
# 评测流程（6 基准 / Avg@8 / Pass@8）与 EOPD 完全相同，只是 MODEL_PATH 指向 OPD 的 checkpoint。

# ---------- 0. 当前改动说明 ----------
# run_eopd.sh 原本写死 METHOD=eopd；现已改为 export METHOD="${METHOD:-eopd}"，
# 因此可直接用 METHOD=opd 覆盖。请先 git pull 拿到该改动。

#   从 DATA_DIR 取 dapo_math17k.parquet 与 math500/test.parquet、按 CKPT_DIR 存 checkpoint。
